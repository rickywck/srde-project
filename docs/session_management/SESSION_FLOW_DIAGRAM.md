## Key Takeaways

1. **Single Agent Instance per run_id** = Conversation continuity
2. **FileSessionManager** = Automatic persistence
3. **agents_cache** = Efficient reuse across requests
4. **run_id** = Session identifier (must be consistent)
5. **Automatic** = No manual save/load needed

The agent "remembers" because:
- Same agent instance is reused
- `agent.messages` contains full history
- FileSessionManager persists to disk
- Sessions survive server restarts
